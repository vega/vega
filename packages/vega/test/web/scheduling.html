To profile: open the DevTools Performance panel and use its reload-and-record button. Compare traces with the scheduling checkbox off and on. Off, dataflow evaluation and canvas rendering run as one long task. On, the run splits into many short tasks that yield via scheduler.yield(), and the main thread stays responsive between them. The total run time appears as the vega scheduled run measure in the Timings track; compare it across traces to see the overhead scheduling adds. Check the second box to simulate browsers without the Prioritized Task Scheduling API (Safari): it deletes globalThis.scheduler before the run, so yields fall back to setTimeout(0) and show up in the trace as timer-fired tasks instead.
